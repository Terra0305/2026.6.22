# AquaGuard AI

조선대학교 AI부트캠프 해커톤 우승작
발표 PPT: https://www.canva.com/design/DAHNQ6zTlEk/c4ndsNDftl_H7CTK8wJUSA/edit
양식장 IoT 센서 데이터를 기반으로 펌프·수조 설비 이상을 실시간으로 감지하고, 관리자에게 직관적인 경고와 AI 의심 분석 결과를 제공하는 모바일 최적화 웹 데모입니다.

## 프로젝트 개요

AquaGuard AI는 양식장 설비 관리자가 현장의 이상 징후를 빠르게 파악할 수 있도록 만든 안전 모니터링 서비스입니다.

수압과 진동 이상 점수를 실시간으로 시각화하고, 정상 → 주의 → 위험 단계 변화에 따라 모바일 화면, 그래프, 알림 목록, 장비 상세 화면이 함께 동기화됩니다. 위험 상황에서는 앱 내부에 긴급 경고 화면이 표시되고, 장비 상세 화면에서는 의심 부위인 베어링 / 모터축을 구조도로 보여줍니다.

## 주요 기능

- 모바일 크기에 최적화된 AquaGuard AI 앱 화면
- 수조 현황, 알림 목록, 장비 상세, 차트, 점검 기록 화면 구현
- 센서 시나리오 시뮬레이터 제공
- `정상 상태` / `마모 발생` 버튼을 통한 실시간 상태 전환
- 수압·진동 이상 점수 실시간 변화
- 정상 → 주의 → 위험 단계별 상태 배지 변경
- 위험 상황 시 앱 내부 긴급 경고 오버레이 표시
- AI 의심 분석 결과 및 의심 부위 구조도 표시
- 정상 상태에서는 의심 분석 결과 자동 숨김
- 같은 브라우저 환경에서 시뮬레이터와 앱 화면 실시간 동기화

## 시연 흐름

1. 왼쪽 브라우저에 시뮬레이터를 연다.
2. 오른쪽 브라우저에 모바일 앱 화면을 연다.
3. 시뮬레이터에서 `마모 발생`을 누른다.
4. 앱 화면의 수압, 진동 점수, 그래프, 상태 배지가 정상 → 주의 → 위험으로 변하는 것을 확인한다.
5. 위험 단계 진입 시 긴급 경고 화면과 알림 목록이 동기화되는 것을 확인한다.
6. 장비 상세 화면에서 AI 의심 분석 결과와 의심 부위 구조도를 확인한다.
7. `정상 상태`를 눌러 센서값이 정상 범위로 회복되는 흐름을 확인한다.

## 실행 방법

```bash
node local-server.mjs
```

실행 후 아래 주소를 같은 호스트와 포트로 열어야 실시간 상태가 공유됩니다.

- 시뮬레이터: `http://localhost:4173/simulator`
- 모바일 앱 화면: `http://localhost:4173/`

## 화면 구성

- `index.html` — 홈 / 실시간 모니터링 화면
- `simulator/index.html` — 해커톤 시연용 센서 시뮬레이터
- `detail.html` — 장비 상세 및 AI 의심 분석 결과
- `chart.html` — 수압 변화 차트 및 분석 화면
- `alerts.html` — 알림 목록
- `tanks.html` — 수조 현황
- `report.html` — 안전 점검 기록
- `account.html`, `settings.html` — 계정 및 설정 화면

## 기술 구성

- HTML
- Tailwind CSS CDN
- Vanilla JavaScript
- LocalStorage
- BroadcastChannel API
- Vercel 정적 배포

## 상태 동기화 방식

`demo-state.js`에서 공통 센서 상태를 관리합니다.

- 시뮬레이터가 상태 변경 이벤트를 발생시킵니다.
- `localStorage`에 현재 상태를 저장합니다.
- `BroadcastChannel`을 통해 같은 브라우저의 다른 화면에 즉시 전달합니다.
- 각 화면은 상태를 구독하고 센서값, 배지, 그래프, 알림 UI를 갱신합니다.
